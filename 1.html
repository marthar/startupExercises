Let's Talk HTML

HTML

Hypertext Markup Language

HTML is a markup language, which means you're annotating (marking up) plain text to give it structure and meaning, similar to the way you would underline or make notes in a book.

HTML "1.0"

  • Original draft from 1992

  • 20 Tags, including: section, h1-h6, p, a, menu, ul, ol, dir, li, dl dt, dd

  • How stuff was displayed was left up to the browser

  • Was more concerned with “Semantic” markup of documents

The Browser War, Round 1

  • Mozilla vs. Microsoft.

  • Started adding features fast and furious

  • Lots of incompatabilities

XHTML

  • Reaction to “looseness” of HTML

  • A XHTML page is a valid XML document

  • In XHTML 4.0 Strict, document won’t render if there’s a mistake in the HTML

  • Boring...Coming down from the W3C

HTML5 and The New Hope

  • Previous versions of HTML came from the IETF (Now Defunct) and W3C

  • 2004: There was a rebellion over XHTML (non-backwards compatible version of HTML)

  • WHATWG Formed (Apple, Mozilla, Opera)

  • 2007: Adopts HTML5, drops XHTML

Let's Review

Tags

HTML tags are element names surrounded by angle brackets like:
< html >

Tags

HTML tags usually come in pairs

<head>   </head>

In the industry we refer to these as start tag and end tag or open tag and close tag

Tags

In HTML you can write just the opening tag and it will often work like:

<h1> Headline
<p> Body text.

Don’t do this. Always use open and close or single tags.

Tags

Tell the browser about your content

  • <html> describes the web page

  • <body> visible page content

  • <div> defines a division or a section

  • <span> group inline-elements together

  • <h1> heading

  • <p> paragraph

  • <em> adds emphasis

  • <header> is header content

  • <nav> is navigational content


Some tags are generic - <div>, <span> Some have “semantic” meaning <h1>, <nav>

Attributes

Attributes

Tell the browser about your content

  • HTML tags/elements can have attributes

  • Attributes are always specified in the start tag

  • Attributes come in name/value pairs like: name="value"

  • <element name = "value">

Attributes

<element  name  = " value ">

Example:

  • links are defined by using an <a> tag

  • you tell the browser what address to link to by adding a href tag

  • <a href = "http://www.massart.edu">

Attributes

Most Common Types

  • CLASS - this defines one or more CSS class names to assign to the tag.

  • ID - this defines a unique identifier that uniquely refers to this one tag.

  • STYLE - this defines the CSS styles to use for this particular element.

  • others: href, src, data-*

Attributes

More Examples

  • <nav id = "top-nav">

  • <img src = "logo.png">

  • <p class = "higlight">

  • <a href = "http://www.massart.edu">

HTML5 Semantic Structure

HTML5 Semantic Tags

Most Common Tags

  • <head> = Meta Info

  • <body> = Site Content

  • <header> = Header (Head area of the site)

  • <nav> = Navigation (Menu/Navigation Area)

  • <section> = Section

  • <article> = Article

  • <footer> = Footer Area

  • <aside> = An area on a side (side bar)

<head> = Meta Info

<head>
  <title>Page Title - important & Required</title>
  <meta name='description' 
        content='Page description - only important for search results page'/>
  
  <meta name='keywords' content='Keywords, not that important'/>

  <link href="stylesheet.css" rel="Stylesheet" type="text/css" />
  ...more stylesheets...

  <script src="somejavascriptfile.js"></script>
  ...more scripts...
</head>

<header>

  • a group of introductory or navigational aids

  • logos, table of contents, search form

<nav>

  • a section of a page that links to other pages or to parts within the page

  • main navigation: home, about, contact, user navigation: login, register, account sidebar navigation: products, blogs, etc.

<section>

  • a group of content, typically with a heading

  • introduction,  
    news items,  
    contact information, 
    chapters, 
    tabbed pages in a tabbed dialog box,  
    numbered sections of a thesis.

<article>

  • Self-contained piece of content

  • forum post,
    magazine article,
    newspaper article,
    blog entry, 
    user-submitted comment, 
    interactive widget, 
    independent item of content.

<footer>

  • represents the footer for the section it applies to. (does not have to mean the bottom of the page)

  • information about its section, 
    who wrote it,  
    links to related documents,  
    copyright data, 
    appendices,  
    indexes,  
    colophons,  
    license agreements.
